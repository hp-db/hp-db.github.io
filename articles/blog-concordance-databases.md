# ヒエラティックとヒエログリフのID体系・関連データベース

## はじめに

古代エジプトの文字には、石碑などに刻まれた**ヒエログリフ（神聖文字）** と、パピルスやオストラコン（陶片・石灰岩片）などに葦ペンで書かれた**ヒエラティック（神官文字）** があります。ヒエラティックはヒエログリフの筆記体（くずし字）にあたり、両者は古い時代からほぼ同時並行で発展してきました。日本語でいえばヒエログリフが「楷書」、ヒエラティックが「草書」のような関係に近いですが、時代が下るにつれて両者の乖離は大きくなり、単純に「くずしただけ」とは言えない独自の発展を遂げています。

HPDB（Hieratische Paläographie DB）は、このヒエラティックの字形を検索するためのデータベースです。ヒエラティックの記号には、メラー番号という独自の番号が振られていますが、同じ記号をヒエログリフとして見ると、ガーディナー番号やUnicodeなど**別の番号体系**で呼ばれます。100年以上にわたって異なる研究者や機関がそれぞれ独自の分類体系を作ってきたためです。

「ID コンコーダンス表」は、これらの異なる番号体系を**一覧で対応づける**ためのツールです。いわば、ヒエラティックとヒエログリフの世界をつなぐ「翻訳表」のようなものです。

---

## なぜ番号体系がこんなに多いのか？

たとえば、「座った男性」の記号 𓀀 を例にとると：

| 体系 | 番号 | 由来 |
|------|------|------|
| メラー番号 | **33/33B** | 1909年のドイツの学者によるヒエラティック分類 |
| ガーディナー番号 | **A1** | 1927年のイギリスの学者によるヒエログリフ分類 |
| Unicode | **U+13000** | 2009年のコンピュータ文字コード |
| Wikidata | **Q117849041** | ウィキペディア関連の知識ベース |

同じ記号なのに、4つも違う番号がある。これが **937個** の記号すべてで起きているわけです。メラー番号はヒエラティック側から、ガーディナー番号はヒエログリフ側からつけた番号なので、番号の振り方がまったく異なります。

## 各ID体系の解説

### 1. メラー番号（Moller No.）── 最古の体系

**作った人：** ゲオルク・メラー（Georg Möller、ドイツの学者）
**いつ：** 1909〜1936年
**対象：** ヒエラティック（神官文字＝ヒエログリフの筆記体）

メラーは、古代エジプトの手書き文字（ヒエラティック）の初の包括的な古書体学を確立した人物です。彼の著書『Hieratische Paläographie』（ヒエラティック古書体学）で、ヒエラティック記号にそれぞれ番号を振りました。HPDBの名前もこの著書に由来しています。本書は全3巻（第1巻1909年、第2巻1910年、第3巻1912年）で、メラーの死後1936年にGrapowによる補遺巻が加わりました。

**特徴：**
- 番号は古いTheinhardt/Lepsius活字体系に基づいており、ガーディナー記号表のような体系的なカテゴリ分類ではない（例：メラー1番＝ガーディナーA26、メラー2番＝A30と、番号順がガーディナーの分類順と一致しない）
- 「8bis」「42B」のように、後から追加された記号や変形には接尾辞がつく（bisはラテン語で「2度目」の意味）
- 「91+575」のような**合字**（複数の記号を組み合わせたもの）にも対応
- **614〜706** あたりの番号は「数字の繰り返し」で使われる（例：614〜622は数字の1〜9を表すストローク記号）

**一言で言うと：** ヒエラティック研究の「元祖」。古い文献を読むときに必須。

---

### 2. ガーディナー記号表（Gardiner Sign List）── 最も広く使われる体系

**作った人：** アラン・ガーディナー（Sir Alan Gardiner、イギリスの学者）
**いつ：** 1927年（『Egyptian Grammar』初版）
**対象：** ヒエログリフ（正式な刻文字体）

ガーディナーは、ヒエログリフをカテゴリ別に整理しました：

| カテゴリ | 内容 | 例 |
|----------|------|-----|
| A | 男性 | A1（座った男性 𓀀） |
| B | 女性 | B1（座った女性 𓁐） |
| D | 体の部分 | D1（頭 𓁶）、D21（口 𓂋） |
| E | 哺乳動物 | E23（ライオン 𓃭） |
| G | 鳥 | G1（ハゲワシ 𓄿）、G17（フクロウ 𓅓） |
| M | 植物 | M17（葦 𓇋） |
| N | 天体・地形 | N5（太陽 𓇳） |
| O | 建物 | O1（家 𓉐） |

**特徴：**
- 「A1」「D21」のように **アルファベット＋数字** の形式
- アルファベットが**カテゴリ**、数字が**その中での通し番号**
- 現在のエジプト学で最も広く使われている
- 「A??」は「ガーディナー表に載っていない記号」を意味する

**一言で言うと：** ヒエログリフの「共通語」。論文を書くときはまずこの番号を使う。

---

### 3. Unicode（ユニコード）── コンピュータの世界の体系

**作った組織：** Unicode Consortium（国際的な文字コード標準化団体）
**いつ：** 2009年（Unicode 5.2でエジプト・ヒエログリフが追加）
**対象：** コンピュータで表示するための文字コード

Unicodeは、世界中の文字をコンピュータで扱うための国際規格です。エジプト・ヒエログリフもここに含まれています。

**特徴：**
- **U+13000〜U+1342F** の範囲に割り当て
- この記事の中で実際にヒエログリフが表示されているのは、Unicodeのおかげ（例：𓀀 𓂋 𓇳）
- Unicode自体には1,071以上のヒエログリフ文字が収録されているが、HPDBが扱う937記号のうちUnicodeに対応するのは628（残りはガーディナー表にない拡張記号や合字など）

**一言で言うと：** ヒエログリフをメールやWebページに書くための番号。

---

### 4. JSesh ── ヒエログリフ入力ツールの体系

**作った人：** Serge Rosmorduc（フランスの研究者）
**対象：** ヒエログリフの電子組版（テキスト入力・表示）

JSeshは、コンピュータ上でヒエログリフを入力・表示するためのソフトウェアです。エジプト学者がよく使います。

**特徴：**
- 基本的にガーディナー番号をそのまま使う（A1、D21など）
- ガーディナー表にない記号にも独自のコードを割り当てている
- 記号の**SVG画像**を生成できる
- 「`sr A21`」のように、記号の転写（読み方）がコードに含まれることもある

**一言で言うと：** ヒエログリフ版の「ワープロ」のための記号コード。

---

### 5. Manuel de Codage（MdC）── テキスト入力の古い標準

**作った人：** Jan Buurman、Nicolas Grimal、Jochen Hallofら（「Informatique et Egyptologie」委員会）
**いつ：** 1984年に委員会発足、1988年に刊行
**対象：** コンピュータでヒエログリフを符号化する方法

Manuel de Codage（マニュエル・ド・コダージュ、「コード化手引き」の意）は、ヒエログリフをテキストファイルで記述するための標準規格です。

**特徴：**
- ガーディナー番号をベースにしつつ、記号の配置・方向・グループ化のルールを追加
- テキストファイルでヒエログリフのレイアウトまで指定できる
- JSeshなどのソフトウェアが、この MdC を入力方式として採用している（JSeshはMdCの「実装」）
- Unicodeの普及により、文字そのものの符号化としての役割は薄れつつあるが、レイアウト指定の規格としては現役

**一言で言うと：** ヒエログリフの「入力・配置コード」。JSeshの裏側で今も動いている。

---

### 6. Hieroglyphica（HG）── ヒエログリフの包括的カタログ

**作った人：** Nicolas Grimal ら
**対象：** 拡張されたヒエログリフの記号一覧

Hieroglyphicaは、ガーディナー表を大幅に拡張した記号カタログです。ガーディナーが763の基本記号をカバーしたのに対し、Hieroglyphicaは数千以上の変形やバリエーションを収録しています（第2版、2000年）。

**特徴：**
- ガーディナー番号と同じ形式（A1、D21など）だが、より多くの記号をカバー
- TSL（Thot Sign List）で参照できる
- 番号の後にカンマと数字がつくことも（例：「4,5」＝ページ4の5番目）

**一言で言うと：** ガーディナー表の「拡張版」カタログ。

---

### 7. IFAO番号 ── フランスの研究所の体系

**作った組織：** Institut francais d'archeologie orientale（フランス東洋考古学研究所、カイロ）
**対象：** IFAOが作成したヒエログリフフォントの記号番号

IFAOはカイロにあるフランスの考古学研究所で、独自のヒエログリフフォントを開発しています。

**特徴：**
- 「4,5」のような **ページ番号,記号番号** の形式
- IFAOの出版物やフォントで使われる
- 個別の記号検索ページはなく、フォント配布ページのみ

**一言で言うと：** IFAO出版物を読むときに出てくる番号。

---

### 8. Wikidata ── リンクトデータの世界

**運営：** ウィキメディア財団
**対象：** あらゆる知識を構造化データとして記述

Wikidataは、ウィキペディアの「裏側」にあるデータベースで、あらゆる概念に固有のID（Qから始まる番号）を割り当てています。

**特徴：**
- 「Q117849041」のようなQから始まるID
- 記号の名前、説明、他のデータベースとの対応関係が記録されている
- HPDBの937記号中533がWikidataにリンク済み
- 多言語の情報にアクセスできる

**一言で言うと：** ヒエログリフの情報をWeb上で繋ぐための「接着剤」。

## 関連データベースの解説

### Thesaurus Linguae Aegyptiae（TLA）
ベルリン・ブランデンブルク科学アカデミーとザクセン科学アカデミーが共同で運営する、**古代エジプト語テキストの総合コーパス**。個々の記号だけでなく、実際のテキストの中でその記号がどう使われているかを調べられます。「辞書」と「用例集」を兼ねたようなもの。

### JSesh
前述のヒエログリフエディタ。**ガーディナー番号を入力すると、きれいなヒエログリフ画像を生成**してくれます。エジプト学の論文執筆では定番のツール。無料で利用可能。

### Unicode Egyptian Hieroglyphs（Unikemet）
Unicodeコンソーシアムが管理するヒエログリフの**公式文字コード表**（PDF）。どの記号にどのコードポイントが割り当てられているか確認できます。

### ORAEC - MdC to Unicode
Manuel de CodageからUnicodeへの**変換ツール**。古いMdC形式で書かれたテキストをUnicodeに変換するときに便利。

### Wikidata - Egyptian Hieroglyphs
ヒエログリフに関するWikidataアイテムの一覧。各記号の**多言語での名称、ガーディナー番号、Unicode、他のデータベースとの対応**などが構造化データとしてまとまっています。

### Hieratic Palaeography（St Andrews）
セント・アンドリュース大学のMark-Jan Nederhofが公開する、古代エジプト文字に関するデジタルツール群。**メラー番号とUnicode/ガーディナーの対応表**を含み、コンコーダンス表の参考データの一つ。

### Thot Sign List（TSL）
ガーディナー番号、Hieroglyphica番号、JSeshコード、Unicodeを**横断的に検索**できるオープンなデジタル記号レパートリー。「この記号の別の番号体系での呼び方は？」という疑問に答えてくれます。

### IFAO - Hieroglyphic Fonts
IFAO（フランス東洋考古学研究所）が提供する**ヒエログリフフォント**。学術出版で使われる高品質なフォントを無料でダウンロードできます。

## ID体系の違いを図で理解する

```
同じ記号「座った男性」𓀀 の各体系での表し方：

メラー (1909)     : 33/33B    ← 通し番号（Theinhardt/Lepsius活字体系に由来）
ガーディナー (1927) : A1        ← カテゴリ＋番号（A＝男性、1番目）
Unicode (2009)     : U+13000   ← 16進数のコードポイント
JSesh              : A1        ← ガーディナー番号を流用
MdC                : A1        ← ガーディナー番号を流用
Hieroglyphica      : A1        ← ガーディナー番号を流用（1,1＝1ページ1番目）
IFAO               : 1,1       ← ページ,記号番号
Wikidata           : Q117849041 ← グローバルID
```

**ポイント：** JSesh、MdC、Hieroglyphicaは基本的にガーディナー番号を踏襲しています。つまり、大きく分けると**メラー番号系（ヒエラティック由来）** と**ガーディナー番号系（ヒエログリフ由来）** と**Unicode系（コンピュータ由来）** と**Wikidata系（Web由来）** の4つのグループがあると考えるとわかりやすいです。コンコーダンス表の最大の価値は、ヒエラティック側（メラー）とヒエログリフ側（ガーディナー）の橋渡しをしてくれることにあります。

## 合字（リガチャ）について

コンコーダンス表には「91+575」「282+196B+327+99」のような複合エントリがあります。これは**複数のヒエログリフを組み合わせた記号**です。

例：
- **91+575** ＝ D21（口 𓂋）＋ X1（パン 𓏏）→ 組み合わせて一つの文字として使う
- **33B+561** ＝ A1（男性 𓀀）＋ Z2（三本線 𓏥）→ 複数形を表す

古代エジプト人は、ヒエログリフを単独でも、組み合わせても使いました。コンコーダンス表はこれらの組み合わせも網羅しています。

## まとめ

| 「何を知りたいか」で使うデータベースが変わる |
|---|
| **ヒエラティック（手書き）の字形を見たい** → メラー番号で HPDB を検索 |
| **対応するヒエログリフを知りたい** → コンコーダンス表でメラー→ガーディナーを変換 |
| **ヒエログリフの基本情報を調べたい** → ガーディナー番号で Wikipedia や TSL を検索 |
| **コンピュータで記号を表示したい** → Unicode を使う |
| **論文にきれいなヒエログリフを入れたい** → JSesh でガーディナー番号を入力 |
| **記号の学術的な詳細データがほしい** → Wikidata の Q番号でアクセス |
| **番号の対応がわからない** → コンコーダンス表で変換！ |

コンコーダンス表は、これらすべての体系を**橋渡し**する役割を果たしています。一つの番号体系しか知らなくても、この表を使えば他のすべての体系での番号がわかります。
